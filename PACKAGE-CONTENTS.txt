INSTALLATION REQUIREMENTS
================================================================================

REQUIRED:
- Node.js 18 or later (https://nodejs.org/)
- PowerShell 7+ (https://github.com/PowerShell/PowerShell/releases)
- AvePoint Fly.Client PowerShell module
- Valid AvePoint Fly API credentials
- Windows 10/11 (tested on Windows 11 Pro 10.0.26200)

RECOMMENDED:
- 4GB RAM minimum
- 500MB free disk space
- Internet connection for API access

================================================================================
CONFIGURATION FILES
================================================================================

The application stores configuration at:
%APPDATA%\FlyMigration\config.json

This file contains:
- AvePoint Fly API URL
- Client ID
- Encrypted client secret (Windows DPAPI)
- Portal URLs
- Customer/tenant information

IMPORTANT: This config file is user-specific and machine-specific.
Do not copy between users or machines.

================================================================================
KNOWN LIMITATIONS
================================================================================

1. Windows only (uses Windows DPAPI for encryption)
2. Requires active AvePoint Fly subscription
3. Migration projects must follow naming convention:
   {CustomerPrefix} - {WorkloadName}
   Example: "Contoso - Exchange", "Contoso - SharePoint"
4. Client secret encryption is non-portable between users/machines

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues:

"Application won't start"
→ Solution: Verify Node.js installed, run INSTALL.bat

"Fly.Client module not found"
→ Solution: Install-Module -Name Fly.Client

"Connection failed"
→ Solution: Verify credentials, click Test Connection

"Dashboard shows no data"
→ Solution: Ensure projects exist in AvePoint Fly with correct naming

"Client secret not saving"
→ Solution: Click Save All button, field shows dots when saved

For detailed troubleshooting, see README.md
