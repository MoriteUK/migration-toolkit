================================================================================
MIGRATION TOOLKIT - QUICK START GUIDE
================================================================================

STEP 1: INSTALL PREREQUISITES
------------------------------
Before running the toolkit, ensure you have:

1. Node.js 18 or later
   Download: https://nodejs.org/
   Verify: Open PowerShell and run: node --version

2. PowerShell 7 or later (pwsh.exe)
   Download: https://github.com/PowerShell/PowerShell/releases
   Verify: Open PowerShell and run: pwsh --version

3. AvePoint Fly.Client PowerShell Module
   Install: Open PowerShell and run: Install-Module -Name Fly.Client
   Verify: Get-Module -Name Fly.Client -ListAvailable


STEP 2: RUN INSTALLATION
-------------------------
1. Double-click INSTALL.bat
2. Wait for installation to complete
3. Press any key to close the window


STEP 3: LAUNCH THE APPLICATION
-------------------------------
Option A: Double-click START.bat
Option B: Open PowerShell in MigrationToolkit-Web folder and run: npm start


STEP 4: CONFIGURE API CREDENTIALS
----------------------------------
1. Click the Settings gear icon (⚙) in the sidebar
2. Go to "Config" tab
3. Enter your AvePoint Fly API credentials:
   - FLY API URL: https://graph.avepointonlineservices.com/fly
   - AOS CLIENT ID: (your client ID)
   - CLIENT SECRET: (your client secret)
   - SECRET EXPIRY: (expiry date)
   - FLY PORTAL URL: https://fly.avepointonlineservices.com/#/project/all-list
4. Click "Test Connection" to verify
5. Click "Save All"


STEP 5: ADD CUSTOMERS
---------------------
1. In Settings, go to "Customer" tab
2. Click "Add Customer" button
3. Enter customer details:
   - Prefix: Short identifier (e.g., "Contoso")
   - Account Name: M365 account email
   - SharePoint Admin URL: https://tenant-admin.sharepoint.com
4. Click "Save All"


STEP 6: VIEW DASHBOARD
-----------------------
1. Click "Home" in the sidebar
2. Select a migration from the dropdown
3. Click "Refresh" to load latest data
4. Click on stat boxes for detailed information


TROUBLESHOOTING
---------------
Problem: Application won't start
Solution: Run INSTALL.bat again, check Node.js is installed

Problem: "Fly.Client module not found"
Solution: Install module: Install-Module -Name Fly.Client

Problem: "Connection failed"
Solution: Verify API credentials, check "Test Connection"

Problem: Dashboard shows no data
Solution: Ensure projects exist in AvePoint Fly with naming format:
          {CustomerPrefix} - {Workload}
          Example: "Contoso - Exchange", "Contoso - SharePoint"

Problem: Client secret not saving
Solution: Make sure to click "Save All" button. Field will show dots
          (••••••••) when a secret is configured.
